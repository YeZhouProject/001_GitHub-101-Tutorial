# GitHub Pages

## What is GitHub Pages?

GitHub Pages is a service provided by GitHub that allows users to publish websites directly from a GitHub repository.

It is commonly used for:

* Personal websites
* Project documentation
* Portfolio websites
* Open-source project pages
* Technical tutorials

Official documentation:

https://docs.github.com/en/pages

---

# How GitHub Pages Works

GitHub Pages converts files stored in a repository into a public website.

The basic workflow:

```
Create Website Content

        ↓

Store Files in GitHub Repository

        ↓

Enable GitHub Pages

        ↓

Website Published Online
```

---

# GitHub Pages and Markdown

Markdown files can become website content.

Example:

```
Repository

├── README.md
├── docs
│   ├── 01-Introduction.md
│   ├── 02-Git-Installation.md
│   └── 03-VSCode.md
```

These documents can be published as a structured tutorial website.

---

# Enabling GitHub Pages

To enable GitHub Pages:

1. Open your GitHub repository.

2. Select:

```
Settings
```

3. Open:

```
Pages
```

4. Choose the publishing source.

Common options:

```
Deploy from a branch
```

5. Select:

```
main branch
```

6. Save the settings.

GitHub will generate a public website address.

# Improving the Tutorial Website

A basic GitHub Pages website works immediately.

However, documentation projects often use themes and static site generators.

Popular tools include:

## Jekyll

Jekyll is a static site generator supported by GitHub Pages.

Official website:

https://jekyllrb.com/

It can create:

* Navigation menus
* Page layouts
* Themes
* Documentation websites
